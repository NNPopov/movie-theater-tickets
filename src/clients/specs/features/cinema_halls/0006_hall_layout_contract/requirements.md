# 0006 · hall_layout_contract — Requirements

## Functional Requirements

| ID | Requirement |
|---|---|
| F1 | The system shall expose a hall's geometry as a single `SeatLayout` value carrying `{ hallId, bounds, screen, zones, seats }`. |
| F2 | The system shall represent every seat as a flat `SeatPlacement` entry with its own `(x, y)` position, with no structural nesting by row. |
| F3 | The system shall express all geometry in layout space (seat-pitch units, origin top-left, y-down) and shall never carry pixel values. |
| F4 | The system shall key each `SeatPlacement` by `SeatId = (row, seatNumber)`, derived from its `row` and `number`. |
| F5 | The system shall expose `SeatLayout.bounds` as an explicit canvas extent, not as a value derived from the seat bounding box. |
| F6 | The system shall expose an explicit `screen` element giving the screen's side and start/end position in layout space. |
| F7 | The system shall expose `zones` as draw-only regions, each carrying `{ id, label, colour, polygon }`, where `colour` is a hex string and `zones` may be empty. |
| F8 | The system shall derive a seat's zone membership solely from its explicit `zoneId`, never from the polygon it visually sits within. |
| F9 | The system shall provide a single `SeatLayoutSource` port exposing `getLayout(hallId) → SeatLayout`, shaped after `GET …/halls/{id}/layout`. |
| F10 | The system shall obtain a `SeatLayout` for a legacy hall by synthesizing one from the existing `List<List<CinemaSeat>>` grid while the backend is mocked. |
| F11 | The synthesizer shall map each `CinemaSeat` to a `SeatPlacement` with `x = columnIndex`, `y = rowIndex`, `w = h = 1`, `rotation = 0`, `zoneId = null`, `row`/`number` from the seat, and `hallId = hall.id`. |
| F12 | The synthesizer shall place `screen` at the top spanning the seat width with `side = top`, `start = (0, -m)`, `end = (C, -m)` where `m = kLegacyLayoutMargin`. |
| F13 | The synthesizer shall compute `bounds` as `x = -m`, `y = -2m`, `width = C + 2m`, `height = R + 3m`, where `R` is the row count and `C` is the maximum seats across rows. |
| F14 | The synthesizer shall emit `zones = []` for legacy halls. |
| F15 | The synthesizer shall map an empty hall (`cinemaSeat == []`) to `seats: []` with `C = 0` and `R = 0` without throwing. |
| F16 | The synthesizer shall map a ragged inner list by each seat's own indices without padding. |
| F17 | The bootstrap adapter shall return `Right(SeatLayout)` produced by the synthesizer when the hall data is obtained successfully. |
| F18 | The bootstrap adapter shall pass a known `Left(Failure)` from `getCinemaHallInfoById` through unchanged, without synthesizing a layout. |
| F19 | The bootstrap adapter shall, on an unexpected exception, return `Left(ServerFailure)` and log the failure rather than rethrow. |
| F20 | The system shall register `SeatLayoutSource` in dependency injection so the port is resolvable, without wiring any consumer to it. |
| F21 | The system shall leave the existing seat grid, booking, reservation, shopping-cart, SignalR status, and pricing behaviour unchanged and produce no user-visible change. |

## Non-functional Requirements

| ID | Requirement |
|---|---|
| N1 | The `SeatLayout` and its parts (`SeatPlacement`, `Zone`, `Screen`, `LayoutBounds`, `LayoutPoint`) shall be immutable value types generated by `freezed` + `json_serializable`, mirroring the existing `movie.dart` pattern. |
| N2 | No file under `domain/layout/` shall import `package:flutter/*`, `package:dio/*`, or any package outside `dartz`/`freezed`/pure Dart. |
| N3 | The contract shall carry no money; `SeatId` and `zoneId` shall be the only seams price or status can key on. |
| N4 | `SeatId` shall be imported from `lib/src/seats/domain/entities/seat_id.dart`, not redeclared in `cinema_halls`. |
| N5 | The `seatId` value shall be a derived getter on `SeatPlacement` and shall not be a serialized JSON field. |
| N6 | `w` and `h` shall default to `1.0` and `rotation` shall default to `0.0` when omitted. |
| N7 | `SeatLayout.fromJson`/`toJson` shall round-trip the contract so the backend can later serve the same shape without a model change. |
| N8 | The legacy synthesizer shall be a pure, Flutter-free top-level function with no I/O, no `getIt`, and no logging. |
| N9 | The legacy margin shall be a named top-level constant `kLegacyLayoutMargin = 1.0`, referenced by both the synthesizer and its tests rather than inline literals. |
| N10 | The bootstrap adapter shall include a catch-all `catch (e, st)` with logging per `agent_docs/error_handling.md` and `CLAUDE.md`. |
| N11 | The bootstrap adapter shall accept an injectable `AppLogger` so tests can verify the log call, defaulting to `getLogger(...)` in production. |
| N12 | The bootstrap adapter shall be marked as explicit throwaway scaffolding with a file-top comment naming its deletion trigger (the backend serving real `SeatLayout`). |
| N13 | No new dependency shall be added to `pubspec.yaml`; all required packages already exist. |
| N14 | The slice shall not migrate the legacy `cinema_halls`/`seats` code (no renaming `*Repo`/`*Bloc`, no `slang`, `retrofit`, or `injectable`); it is additive only. |
| N15 | The synthesizer shall have exhaustive mapping-parity unit tests, including the three seeded shapes locking seat counts 616 / 378 / 180, per `agent_docs/testing.md`. |
| N16 | Model invariant unit tests shall cover `seatId` derivation, defaults, zone-membership-vs-polygon independence, value equality, and JSON round-trip. |
| N17 | Adapter unit tests using `mocktail` shall cover the success, pass-through `Left`, and unexpected-exception (with verified logging) paths. |
| N18 | No Cubit, widget, or outside-in test shall be added, as the slice introduces no bloc, UI, or user-visible behaviour. |
| N19 | The only file modified outside the new contract/test files shall be `lib/injection_container.dart` (one DI registration). |

## Out of scope

- The free-form `CustomPaint`/`InteractiveViewer` renderer and any pixel or canvas-space code (P5).
- The live `PriceBloc` and any per-`SeatId` price or status overlay over SignalR.
- The admin hall editor (P6).
- Any backend change — variable seats-per-row storage, persisted geometry, or the real `GET …/halls/{id}/layout` endpoint.
- Zoom/pan, zone rendering, seat orientation, and curved/staggered drawing.
- Migrating `cinema_halls`/`seats` to the target stack.
- Wiring money or pricing into the layout.
