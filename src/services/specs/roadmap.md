# Roadmap — BookingManagement service slices

Global index of slices for the `.NET` BookingManagement service. `NNNN` is a global
zero-padded counter (max existing + 1), not per-aggregate. This file is **owned by
`/to-prd`**; other skills read it but never write to it.

States: Started · Planned · Formalized · Validated · Test-specified · Red gate set ·
Complete. (See `agent_docs/spec_workflow.md`.)

| 0002 | platform | content_not_found_404 | Complete | ADR-002 step 2: flip `ContentNotFoundException` `204 → 404` centrally, with empty-state carve-outs (`current` cart ⇒ `204`, movie sessions list ⇒ `200 []`). Server-only; Flutter client deferred. |

| 0008 | platform | endpoint_input_guards | Complete | ADR-002 **endpoint-helper tail** (the deferred "endpoint bare-throws" from `0005`–`0007`): replace the two bare `throw new Exception(...)` guards on the ShoppingCart endpoints with specific typed exceptions already mapped by `CustomExceptionHandler` — malformed `X-Idempotency-Key` ⇒ `DomainValidationException` ⇒ **400**; non-Guid/missing `nameidentifier` claim in `GetClientId` ⇒ `UnauthorizedAccessException` ⇒ **401**. | 0009 | platform | cart_ownership_authorization | Started | ADR-003 slice 1: build the central object-level authorization mechanism (IDOR/BOLA, OWASP API1:2023) and apply it to two assigned-cart operations. New `CartOwnershipBehaviour` (MediatR pipeline) over requests implementing a new `ICartScopedRequest` marker (exposes `ShoppingCartId`); loads the cart and applies the two-mode rule — anonymous cart (`ClientId == Guid.Empty`) ⇒ pass (capability preserved), assigned cart ⇒ caller must be authenticated and `cart.ClientId == caller`, else **throw `ForbiddenAccessException`** ⇒ **403**; cart not-found ⇒ pass (handler keeps the 404). Tests = behaviour gate (RED) + central 403-mapping pin + `ICurrentUser` unit + **first `WebApplicationFactory<Program>` end-to-end harness** in the repo. Defers other entry points + hub, `assignclient` hardening, HashId/logging hygiene, the 403-vs-404 disclosure decision, ADR-003 acceptance. Prereq (wrong-owner bug) already fixed in `0003`. |
